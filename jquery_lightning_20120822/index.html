Legacy jQuery

  • 2 JavaScript files
  • ~350 lines of code per file
  • click handlers galore
  • $('.link').live('click', ...)
  • absolute lack of code organization
  • DOMception

Legacy Design

  • delivered HTML fragments over the wire
  • custom jQuery UI stylesheet
  • arbitrary values used for padding, margins, etc.
  • customized UI for specific users: many edge cases

New Code: Backbone.js

  • Breaks down across Models, Views, etc.
  • Breaks out client API from server API
  • Scoping DOM events is dramatically easier
  • 57 files, ~70 lines of code per file
  • There's More Than One Way To Do It

New Code: Twitter Bootstrap

  • Loving the Whitespace
  • Formalized padding, margins, colors etc. with variables (LESS/SCSS)
  • Lightweight jQuery components replace jQuery UI widgets

Rails 3

  • Asset Pipeline enables use of CoffeeScript, SCSS
  • Plugins (gems) available for integrating Backbone.js, Twitter Bootstrap
  • Easy to integrate Jasmine specs from command line
  • templates used to be in app/views, are now in app/assets/javascripts/backbone/templates

Was it worth it?

  • 5 months of development time
  • 10x performance gains
  • fantastic feedback
  • fewer bugs than previous efforts
  • it's freakin' gorgeous
